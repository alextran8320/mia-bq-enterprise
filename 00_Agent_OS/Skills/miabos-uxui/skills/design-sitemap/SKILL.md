# Design Sitemap

## Purpose

Materialize the structural bridge between feature logic and screen-level UX handoff so the UI can be broken into screens without losing end-to-end task flow.

## Quality Checks

- [ ] Every in-scope feature maps to one or more screens
- [ ] Routes / drawers / modals are represented
- [ ] Entry points and transitions are explicit
- [ ] One dominant job per route is preserved
