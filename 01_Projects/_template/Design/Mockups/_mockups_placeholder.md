## What Goes Here

This folder contains all UI mockups for the project. Mockups must be created by **A06 UI/UX Agent** and signed off by the Business Owner before Gate 2B can pass.

### Required Screens (per Gate 2B)
Every project must have mockups for:
- [ ] Home / Dashboard screen
- [ ] Key interaction screen (primary user action)
- [ ] Form screen (data entry)
- [ ] List / Table screen (data display)
- [ ] Empty state screen
- [ ] Error state screen
- [ ] Loading state screen

### File Naming Convention
```
[ScreenName]_[State]_v[Version].md
```
Examples:
- `Dashboard_Default_v1.md`
- `OrderForm_Empty_v1.md`
- `OrderList_Loaded_v2.md`
- `Login_Error_v1.md`

### Mockup Format
Each mockup file should include:
1. **Screen name and purpose**
2. **Design direction** (which of the 3 directions was approved at Gate 2A)
3. **Component inventory** (list of UI components used)
4. **Interaction notes** (hover states, click targets, transitions)
5. **Responsive breakpoints** (if applicable)
6. **Fidelity score target** (≥ 8/10 vs final build)

---

## Gate 2B Checklist (A06 must verify before sign-off)

- [ ] All required screens have mockups
- [ ] Mockups follow the approved design direction from Gate 2A
- [ ] Design System is consistent across all screens
- [ ] Business Owner has reviewed and approved
- [ ] Fidelity target (≥ 8/10) is achievable from these mockups
