## Deprecation Notice

`Feature Brief` is no longer a legal planning artifact in the MIABOS operating system.

Use this chain instead:

`PRD -> Product Backlog (User Stories) -> Sprint Backlog -> User Story -> Feature SRS -> UXUI -> Subtask Board -> FE Preview -> Review / Enhance -> Integration Spec -> BE -> FE Integration -> QA/UAT`

## Replacement Rule

- Use `PRD` for business alignment.
- Use `User Story` for the delivery slice and planning context.
- Use `Feature SRS` for the first canonical feature-behavior contract.

## Historical Note

Old session logs or legacy artifacts may still mention `Feature Brief`. Treat those references as historical only, not as current process law.
